## Pick the right control

| Need | Component | Notes |
|------|-----------|-------|
| Single date in a form field | DateInput | Input + dropdown calendar |
| Date + time (meetings, deadlines) | DateInput with timePicker | Stays open until time is set |
| Range inside a modal or filter panel | DateRangePicker | Standalone dual-month calendar |
| Range with formatted text field | DateRangeInput | Input pair + shared picker |

All date components live under form controls in Core. Import from `@once-ui-system/core` and mark the parent file `"use client"` when you hold state.

## Single date with DateInput

DateInput combines an Input trigger with a dropdown DatePicker. Pass a stable `id`, a `label`, a `Date` value, and an `onChange` handler.

```text
const [date, setDate] = useState<Date | undefined>();

<DateInput
  id="event-date"
  label="Event date"
  value={date}
  onChange={setDate}
/>
```

The input displays a formatted locale string. Clicking or focusing opens the calendar. Selecting a day calls `onChange` and closes the dropdown (unless `timePicker` is enabled).

### Time selection

Set `timePicker` when users need hour and minute precision â€” scheduling calls, publishing windows, or SLA timers.

```text
<DateInput
  id="meeting-time"
  label="Meeting start"
  value={start}
  onChange={setStart}
  timePicker
/>
```

With `timePicker`, the dropdown stays open after the day is picked so the user can adjust hours and minutes.

### Bounds and validation

Pass `minDate` and `maxDate` to block out-of-range days in the calendar. Pair with Input `error` text when server validation fails â€” the picker prevents most mistakes client-side, but API errors still need a message.

## Date ranges

For analytics filters or booking flows, use a range shape:

```text
{ startDate: Date | undefined, endDate: Date | undefined }
```

**DateRangePicker** embeds directly in a Column or Dialog â€” no input field. Ideal for filter sidebars and booking modals where the calendar is always visible.

**DateRangeInput** adds labeled inputs that open the same range picker in a dropdown. Use it when space is tight and the range is secondary until expanded.

Highlight behavior: the first click sets `startDate`, the second sets `endDate`. Days between get a range highlight. Clicking a third time restarts selection.

## Layout in forms

Stack date fields in a Column with `gap="16"`. For "from / to" pairs in one row, use a Row with `gap="12"` and `s={{ direction: "column" }}` so fields stack on mobile.

Put scheduling fields after related text inputs (title, description) and before submit actions. If the range drives other filters (team, status), place the picker above the data table.

## Write this / not this

| âś… Once UI way | âťŚ Common mistake |
|----------------|-------------------|
| Controlled `Date` state in parent | Uncontrolled native `type="date"` inputs |
| `timePicker` only when time matters | Always showing hour/minute pickers |
| `minDate` / `maxDate` on the picker | Letting users pick impossible ranges |
| DateRangePicker in a filter panel | Tiny native inputs with no calendar |

- [ ] Error text shown when server rejects a date
