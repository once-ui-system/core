The `OgCard` component displays rich link previews for URLs by fetching and rendering Open Graph metadata. It's perfect for showcasing external links with preview images, titles, and descriptions.

## Required API routes

To use `OgCard` with automatic Open Graph fetching, your application must expose the following API routes.

These routes are used internally by the component to fetch metadata and safely load external assets.

### 1. OG metadata fetch

`/api/og/fetch`

This endpoint retrieves Open Graph metadata (title, description, image, favicon) from a given URL.

It is used when `OgCard` receives only a `url` prop.

You can refer to the implementation in:
- `handleOGFetch.mdx`

---

### 2. OG image proxy

`/api/og/proxy`

This endpoint proxies external images to avoid CORS restrictions when rendering preview images inside the card.

You can refer to the implementation in:
- `handleOGProxy.mdx`

---

### 3. (Optional) OG generate endpoint

`/api/og/generate`

Used in advanced setups where OG images are dynamically generated server-side.

This is optional depending on your configuration.

## Service configuration

Use `serviceConfig` to point `OgCard` at custom API routes or to transform proxied image and favicon URLs. Pass pre-fetched data with `ogData` to skip the fetch entirely.

<CodeBlock
  marginTop="16"
  marginBottom="24"
  codeHeight={24}
  lineNumbers
  codes={[
    {
      code:
`<OgCard
  url="https://example.com"
  serviceConfig={{
    fetchOgUrl: '/api/og/fetch',
    proxyOgUrl: '/api/og/proxy',
    proxyImageUrl: (url) => \`/api/og/proxy?url=\${encodeURIComponent(url)}\`,
    proxyFaviconUrl: (url) => \`/api/og/proxy?url=\${encodeURIComponent(url)}\`,
  }}
/>

<OgCard
  ogData={{
    url: 'https://example.com',
    title: 'Example',
    description: 'Pre-fetched metadata',
    image: 'https://example.com/og.png',
    faviconUrl: 'https://example.com/favicon.ico',
  }}
/>`,
      language: "tsx",
      label: "Service configuration"
    }
  ]}
/>

## Notes

- The component automatically fetches favicons for the domain
- Images are proxied through an API endpoint to avoid CORS issues
- You can overwrite the default props by providing static values
- You can set any value to `false` to disable it
