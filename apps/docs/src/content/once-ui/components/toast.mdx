The `Toast` component displays temporary notification messages that automatically dismiss after 6 seconds. It's typically used through the [ToastProvider](/once-ui/contexts/toastProvider) context, but can also be used standalone.

<Feedback title="Using Toasts" description={<>For most use cases, use the <InlineCode>useToast</InlineCode> hook from <SmartLink href="/once-ui/contexts/toastProvider">ToastProvider</SmartLink> instead of rendering Toast components directly. The provider handles positioning, stacking, and lifecycle management automatically.</>}/>

## Variants

Toast supports two variants: `success` for positive feedback and `danger` for errors or warnings.

## With actions

## Toaster

<Feedback marginBottom="24" title="Automatic rendering" description={<>You don't need to manually render the Toaster component. It's automatically included when you wrap your app with <InlineCode>ToastProvider</InlineCode>. See the <SmartLink href="/once-ui/contexts/toastProvider">ToastProvider documentation</SmartLink> for setup instructions.</>}/>

The `Toaster` component manages multiple toast notifications, handling positioning, stacking animations, and portal rendering. It's automatically included when using `ToastProvider`.

The Toaster:
- Renders toasts in a fixed position at the bottom center of the screen
- Stacks multiple toasts with scale and opacity animations
- Uses React portals to render outside the normal DOM hierarchy
- Limits width to 32rem for optimal readability

## Accessibility

Toast components include proper ARIA attributes:
- `role="alert"` for screen reader announcements
- `aria-live="assertive"` for immediate notification
- Keyboard-accessible close button with tooltip
